# Glossário

Termos usados na plataforma e neste manual.

| Termo | O que significa |
|-------|-----------------|
| **Administrador** | Conta com acesso total ao painel de gestão. |
| **Usuário** | Membro da rede (gratuito ou assinante), sem acesso às ferramentas de gestão. |
| **Papel (role)** | Define o nível de acesso da conta: Administrador ou Usuário. |
| **PF — Pessoa Física** | Conta de uma pessoa. |
| **PJ — Pessoa Jurídica** | Conta de uma organização/empresa. |
| **Apoiador** | Usuário com assinatura ativa (paga). |
| **Capability / Permissão** | Uma ação que o sistema controla (ex.: "ler a biblioteca", "gerenciar usuários"). |
| **Plano base** | Plano gratuito estrutural; define o que todo usuário logado pode fazer. Não pode ser arquivado. |
| **Plano pago** | Assinatura que libera permissões extras (PF ou PJ). |
| **Assinatura** | Vínculo de um usuário a um plano, com status (ativa, pendente, cancelada, expirada). |
| **Zotero** | Gerenciador de referências bibliográficas; fonte da biblioteca virtual. |
| **Item local** | Item da biblioteca cadastrado manualmente no painel (não vindo do Zotero). |
| **Curadoria** | Ajustes editoriais do admin sobre um item: tags, capa, torná-lo público. |
| **Estatuto e Atas** | Seção pública especial da biblioteca (subconjunto dos itens públicos). |
| **Modalidade** | Formato de um evento/seminário: Online ou Presencial. |
| **Rascunho / Publicada** | Situação de um conteúdo: invisível ao público / visível no site. |
| **Pix** | Pagamento instantâneo brasileiro. |
| **Boleto** | Boleto bancário; pode levar alguns dias para compensar. |
| **Mercado Pago** | Provedor de pagamentos ativo (cartão, Pix, boleto). |
| **PayPal** | Provedor de pagamentos atualmente **desativado**. |
| **Mercado Pago / PayPal "sandbox"** | Ambiente de teste do provedor (não cobra de verdade). |
| **Template de e-mail** | Modelo de um e-mail automático (assunto + corpo) enviado pelo sistema. |
| **Placeholder / variável** | Trecho de um template substituído por dados reais no envio (ex.: nome do usuário). |
| **Fluxo de Segurança** | E-mails críticos (redefinição de senha, verificação) que podem ser editados mas nunca desativados. |
| **Rede de Apoiadores** | Diretório público de profissionais que optaram por aparecer. |
| **Membro / Vínculo (PJ)** | Pessoa associada a uma organização, com status (pendente, ativo, revogado). |
| **Soft-delete (exclusão suave)** | Remoção que oculta o item mas o preserva no banco para auditoria/recuperação. |
| **Sessão** | Período em que você fica logado (24 horas); depois é preciso entrar de novo. |
| **OpenAPI / Swagger** | Documentação técnica padronizada da API (para desenvolvedores). |
